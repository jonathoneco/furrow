## Part 2: Furrow (~7 min)

### 2.1 The knowledge layer — what furrow accumulates

> "Furrow maintains a knowledge base about the project. Let me show you what that looks like."

**Todos** — the backlog:
```bash
cat .furrow/almanac/todos.yaml | head -40
```

> "Structured TODOs with context, effort estimates, status. Not just titles — full context for an AI agent to pick up and work on."

**Roadmap** — generated from todos:
```bash
cat .furrow/almanac/roadmap.md
```

> "This was generated by running `/furrow:triage`. It reads all the TODOs, builds a dependency graph, and organizes them into phases with ordering."

**Rationale** — why things exist:
```bash
cat .furrow/almanac/rationale.yaml | head -30
```

> "Every component has a `exists_because` and `delete_when` — so we know why it's here and when to remove it."

### 2.2 Row lifecycle — a completed piece of work

> "Let me show you what a completed work unit looks like."

```bash
cat .furrow/rows/quality-and-rules/state.json | python3 -m json.tool
```

> "This went through all 7 steps: ideate, research, plan, spec, decompose, implement, review. Six deliverables across 3 parallel waves, zero corrections. Every gate transition has evidence."

```bash
cat .furrow/rows/quality-and-rules/definition.yaml
```

> "It starts with a definition — objective, deliverables with acceptance criteria, constraints. This is the contract."

### 2.3 furrow:next — what's next?

> "So how do I pick up new work? Furrow generates handoff prompts from the roadmap."

```bash
cat .furrow/demo/next-prompt.txt
```

> "This is the output of `/furrow:next`. It reads the roadmap, finds the next unstarted phase, and generates a self-contained prompt I can paste into a new Claude Code session. It includes the `/work` command to run, the scope, source TODOs, and key files."

### 2.4 The flow — starting work (explain, don't run live)

> "When I run that `/work` command, it kicks off the 7-step flow:"

1. **Ideate** — brainstorm, challenge premises, define the work contract
2. **Research** — investigate prior art, architecture options
3. **Plan** — architecture decisions, execution strategy
4. **Spec** — implementation-ready specs with test scenarios
5. **Decompose** — break into waves with specialist assignments
6. **Implement** — specialists execute within file ownership boundaries
7. **Review** — structured evaluation against acceptance criteria

> "Each step has gates — you can't skip ahead. The agent accumulates learnings, and when it's done, the knowledge feeds back into the almanac."
