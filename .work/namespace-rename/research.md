The CLAUDE.md copy is generated by install.sh, so install.sh is the source of truth. But manual edits to CLAUDE.md can drift. **Action**: Ensure install.sh template is the single source; re-run install to sync.

2. **HARNESS_ROOT initialization in 20+ files**: Every hook/script has `HARNESS_ROOT="$(cd "$SCRIPT_DIR/.." && pwd)"`. This is copy-pasted, not sourced from a common file. **Action**: Consider sourcing from `hooks/lib/common.sh` or a new `scripts/lib/init.sh`. 5. **Log prefix already centralized**: `[harness:warning]`/`[harness:error]` defined only in common.sh. Good — single point of change.

### Dedup Recommendation

Address items 1-2 during rename. Item 1 is a real risk (command table drift). Item 2 reduces rename blast radius if fixed first (change one file instead of 20+). Items 3-5 are low priority.

## Deliverable 2: cross-platform-compatibility

### Correction from Ideation

The ideation phase estimated 11 scripts with `(( ))` and 3 with `[[ ]]` in `#!/bin/sh` files. **The portability audit found zero shebang/feature mismatches.** All `#!/bin/sh` scripts use POSIX-compliant constructs only. The earlier count appears to have been inaccurate.

### Actual Issues Found

**Critical (1):**
- `readlink -f` in `install.sh` (lines 62-63, 327-328) — GNU-only. BSD `readlink` doesn't support `-f`. Current fallback (`|| readlink "$_dst"`) is incomplete (only one level of resolution). Fix: portable `_canonicalize_path()` function or `cd && pwd` approach.

**Moderate (1):**
- `expr "$_updated" \> "$_best_ts"` in `hooks/lib/common.sh` (line 34) — string comparison via `expr` is locale-dependent. Works in practice for ISO 8601 timestamps but not guaranteed. Fix: use POSIX `[ "$a" \> "$b" ]` syntax.

**Info (shellcheck):**
- SC2034: Unused `SCRIPT_DIR` in `scripts/triage-todos.sh:19`
- SC2294: `eval` in `scripts/harness-doctor.sh:328`
- SC1091: Dynamic source paths (12 instances, expected)

### Already Safe

- All `$((expr))` arithmetic is POSIX
- All `[ ]` conditionals are POSIX
- `printf "%b"`, `mktemp -d`, `grep -E`, `sed` patterns are portable
- No `sed -i`, `grep -P`, `stat`, `sort -V`, `find -printf`, `xargs -r`
- No `/dev/fd/` paths, no `$RANDOM` in POSIX scripts
- Bash-only features (`BASH_SOURCE`) confined to `#!/bin/bash` test scripts

### Scope Reduction

Given only 2 real issues, deliverable 2 is much smaller than anticipated:
1. Fix `readlink -f` portability in install.sh
2. Fix `expr` comparison in common.sh
3. Run shellcheck validation
4. Document residual risk (no macOS runtime testing)
