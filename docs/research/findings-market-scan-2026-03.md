## Ecosystem Observations

### 1. The "plugin marketplace" era has arrived
Anthropic themselves shipped `claude-plugins-official` (15.3k stars). The ecosystem has exploded from scattered CLAUDE.md configs into installable plugins, skills, and hooks. Multiple competing marketplaces exist:
- Anthropic's official directory
- VoltAgent's collections (skills + subagents)
- antigravity-awesome-skills
- claude-night-market
- ccplugins/awesome-claude-code-plugins

### 2. Cross-agent compatibility is the new norm
Tools increasingly target Claude Code + Codex CLI + Gemini CLI + OpenCode + Cursor. The "skills" format has become a de facto standard that works across agents. Ruler (2.6k stars) explicitly addresses cross-agent rule portability.

### 3. Multi-agent orchestration is the hot category
tmux-based parallelism is the dominant pattern: cmux, multi-agent-shogun, agent-viewer, claude_code_agent_farm, overstory, agor, Crystal/Nimbalyst. Git worktrees are the standard isolation mechanism for parallel agents.

### 4. The "harness" concept has gone mainstream
The term "agent harness" now appears frequently. `everything-claude-code` (116k stars) describes itself as "the agent harness performance optimization system." Multiple repos use the harness metaphor.

### 5. Context engineering is the dominant philosophy
"Context engineering" has replaced "prompt engineering" in the ecosystem vocabulary. Multiple dedicated repos, skills collections, and frameworks center on this concept.

### 6. Desktop GUIs are wrapping terminal agents
cc-switch (35k), opcode (21k), superset (8.2k), AionUi (20k) all provide GUI interfaces over terminal-based coding agents, suggesting terminal-only is a UX gap being filled.

### 7. Skills have eclipsed CLAUDE.md as the primary extension mechanism
The shift from "config repos" to "installable skills" happened in late 2025. The top individual skill (planning-with-files, 17.6k stars) has more traction than most framework repos.

### 8. Eval remains extremely underserved
langchain-ai/claude-code-evals (59 stars) is the most notable eval-specific project, and it's just an ablation study. No framework has first-class eval infrastructure. This is a wide-open gap.

### 9. Hook tooling is maturing but fragmented
SDKs exist in Python, TypeScript, PHP, Ruby, Swift, and Go — but no dominant cross-language standard. disler's hooks mastery (3.4k stars) and Continuous-Claude-v3 (3.6k stars) are the most significant hook-centric projects.

### 10. The scale has shifted dramatically
Projects in this space routinely hit 10k-100k+ stars. The ecosystem has moved from hobbyist config-sharing to production tooling with real adoption. Star counts from even 6 months ago are no longer representative.

## Key Gaps for work-harness-v2

1. **Eval-first design**: No existing tool treats eval as a first-class concern. All are "workflow first, eval if you get to it."
2. **Dual-mode (interactive + autonomous)**: Most tools target either Claude Code CLI (interactive) or Agent SDK (programmatic), not both.
3. **Thin convention layer**: Most frameworks are maximalist (100+ skills, 50+ agents). A disciplined 20-30 file approach would differentiate.
4. **Designed to shrink**: Every tool adds capabilities. None are designed to shed features as platforms absorb them.
5. **First-party primitive alignment**: Most tools reimplement what Claude Code now provides natively (teams, tasks, worktrees). A harness that leverages rather than replaces platform primitives would be novel.
