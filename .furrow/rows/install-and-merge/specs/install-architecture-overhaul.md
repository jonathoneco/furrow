# Spec: install-architecture-overhaul

Wave-1 deliverable of row `install-and-merge`. Establishes the install-artifact
trust boundary: source-repo detection, XDG state quarantine, hook-cascade
protection via a minimal common lib, a standalone `frw rescue`, pre-commit
guards against contamination, and determinism for merge-conflict-prone data
files. Grounded in `research/r1-install-mutations.md`, `r2-commonsh-split.md`,
`r3-boundary-evidence.md`, `r4-rescue-prior-art.md` and `architecture-decisions.md`
AD-1 through AD-9.

## Interface Contract

### `.furrow/SOURCE_REPO` — sentinel marker

- **Path**: `/.furrow/SOURCE_REPO` (tracked; committed in source repo ONLY).
- **Format**: plain UTF-8 text, single line, terminated by `\n`.
- **Exact contents** (byte-for-byte):
  ```
  This file marks the Furrow source repository. install.sh refuses to copy this file into consumer projects. Do not remove.
  ```
- **Consumers**:
  - `install.sh` — on detect, skip the consumer-side symlink bootstrap (no self-symlinking).
  - `bin/frw.d/install.sh` — on detect, branch to source-hosting mode (regenerate specialist/rules symlinks with `../../` targets; no gitignore append; no `.bak` generation).
- **Producer**: landed in step 1a as a single commit. Never regenerated by install.
- **Refuse-copy rule**: `install.sh` MUST `exit 2` with stderr
  `[furrow:error] refusing to copy .furrow/SOURCE_REPO into consumer project` if
  source == target and target carries no `.furrow/SOURCE_REPO` but the copy
  loop would have written one.

### `bin/frw.d/lib/common-minimal.sh` — hook-safe library

Landed in step 1c as NEW file (additive; `common.sh` untouched until step 1f).
Sourced by ALL hooks in `bin/frw.d/hooks/*.sh`. Depends only on POSIX sh + `jq`.

Exported functions (the 8 hook-safe functions from `research/r2-commonsh-split.md §4`; R2 proposed 7 core + `is_row_file` path-test, preserved here):

| Signature | Purpose |
|-----------|---------|
| `log_warning <msg>` — prints `[furrow:warning] <msg>` to stderr; returns 0 | warn-only, no mutation |
| `log_error <msg>` — prints `[furrow:error] <msg>` to stderr; returns 0 | error-only, no mutation |
| `find_active_row` — prints absolute path of most-recently-updated unarchived row to stdout; returns 1 if none | jq over `.furrow/rows/*/state.json` |
| `find_focused_row` — prints focused row path (fallback: find_active_row); returns 1 if none | reads `.furrow/.focused` |
| `read_state_field <row-dir> <jq-path>` — prints field value to stdout; returns jq's exit code | jq extractor, no mutation |
| `row_name <row-dir>` — prints basename of row dir to stdout; returns 0 | pure string op |
| `extract_row_from_path <file-path>` — prints enclosing row dir or empty; returns 0 | string op + existence check |
| `is_row_file <path>` — returns 0 if path is inside `.furrow/rows/`, else 1 | string prefix test |

**Constraint**: total LOC <= 120 after copyright header. No markdown editing, no
focus mutation, no yq calls, no subshell-heavy constructs. Bytewise-identical
function bodies to `common.sh` pre-split (verified by step-1f diff).

**Non-contents** (stay in `common.sh` only): `read_definition_field`,
`current_step`, `step_status`, `has_passing_gate`, `extract_md_section`,
`replace_md_section`, `set_focus`, `clear_focus`.

### `bin/frw.d/scripts/rescue.sh` — standalone repair tool

CLI (per AD-2, `research/r4-rescue-prior-art.md` §"Implied design"):

```
rescue.sh [--apply] [--file <path>] [--baseline-check]
```

- `--file <path>` — target to repair. Default: `bin/frw.d/lib/common.sh`.
- `--apply` — write the restoration. Without it, diagnose-only (print plan).
- `--baseline-check` — diff bundled baseline against live `common-minimal.sh`; exit 0 if match, 3 if drift. Used by CI (step 1m).

**Argument-parsing style**: follow `bin/frw.d/scripts/doctor.sh` — POSIX
`while/case` loop, no `getopts`, unknown flags exit 2 with usage on stderr.

**Recovery algorithm** (per `r4-rescue-prior-art.md` §"Implied design"):
1. If target parses (`sh -n <target>`), print `OK: no rescue needed` to stdout, exit 0.
2. Else if `git show HEAD:<target>` succeeds AND parses, that is the candidate.
3. Else unpack bundled-baseline heredoc, parse-check, use as candidate.
4. With `--apply`: write candidate to `<target>` atomically (tmp + mv).
5. Without: print unified diff of current vs candidate to stdout, exit 0.

**Exit codes**:
| Code | Meaning |
|------|---------|
| 0 | nothing to do / rescue applied / diagnose-only printed plan |
| 1 | target missing AND no usable baseline |
| 2 | usage error (unknown flag, missing required arg) — matches the "unknown flags exit 2" rule in the arg-parsing style above |
| 3 | `--baseline-check` found bundled baseline differs from live `common-minimal.sh` |
| 4 | rescue wrote target but result failed `sh -n` parse check (post-write verification failure) |

**Bundled-baseline format**: single heredoc `<<'FURROW_BASELINE_COMMON_MINIMAL'`
… `FURROW_BASELINE_COMMON_MINIMAL` near the end of `rescue.sh`. The heredoc
body is a bytewise copy of `bin/frw.d/lib/common-minimal.sh`. A comment above
it declares:
`# Baseline frozen from common-minimal.sh; CI --baseline-check guards drift.`

**Sourcing constraint** (AD-2, definition AC 6): rescue.sh MUST NOT source
`common.sh` or `common-minimal.sh`. Enforced by a literal-grep assertion in
step-1e test (forbidden patterns: `. .*common.sh`, `source .*common.sh`,
`. .*common-minimal.sh`, `source .*common-minimal.sh`).

**Callers**:
- `bin/frw` dispatcher gains a `rescue` case that execs `rescue.sh "$@"`.
- Operator may invoke `./bin/frw.d/scripts/rescue.sh` directly if dispatcher itself is broken (per `r4-rescue-prior-art.md` §"Rescue MUST").

**stdin/stdout**: ignores stdin. Diffs and status messages to stdout. Warnings
and errors to stderr via inline `printf '[furrow:%s] %s\n' warning "$msg" >&2`
(does NOT reuse `log_warning` — rescue depends on nothing).

### Pre-commit hooks — contamination guards

Step 1g. Two hooks placed under `bin/frw.d/hooks/`:

1. **`bin/frw.d/hooks/pre-commit-typechange.sh`**
   - **Hook event**: git `pre-commit`, wired via `.git/hooks/pre-commit` dispatcher installed by `bin/frw.d/install.sh`.
   - **Block condition**: any file in `git diff --cached --name-only --diff-filter=T` matching the glob set `{bin/alm, bin/rws, bin/sds, .claude/rules/*}` whose NEW type is symlink (`git diff --cached --raw` mode goes from `100644` to `120000`).
   - **Exit code**: 1 on block, 0 on pass.
   - **Stderr**: `[furrow:warning] pre-commit: refusing type-change -> symlink on <path> (see docs/architecture/self-hosting.md)`. Emits even when `--no-verify` is used (the hook's own check still fires; git bypasses only the exit code).

2. **`bin/frw.d/hooks/pre-commit-bakfiles.sh`**
   - **Hook event**: same pre-commit dispatcher.
   - **Block condition**: any path in `git diff --cached --name-only` matching `bin/*.bak` OR `.claude/rules/*.bak`.
   - **Exit code**: 1 on block, 0 on pass.
   - **Stderr**: `[furrow:warning] pre-commit: refusing to stage install-artifact <path>; move to $XDG_STATE_HOME/furrow/`.

**`[furrow:warning]` stderr format** (exact, matches `common-minimal.sh`):
`[furrow:warning] <category>: <message>\n`

`--no-verify` trail (per AD-9): both hooks emit their warning BEFORE checking
`$FURROW_HOOK_BYPASS` or exiting. Under `git commit --no-verify`, the hook
body is not invoked by git at all — but CI's contamination-check (step 1m) is
the belt-and-suspenders layer that catches the push.

### `install-state.json` — schema (AD-3)

Written at `$XDG_STATE_HOME/furrow/{repo-slug}/install-state.json` (default
`~/.local/state/furrow/{repo-slug}/install-state.json`). Created/updated by
`bin/frw.d/install.sh` in step 1j. `repo-slug` derivation (**normalized**):

```
slug = basename "$(git rev-parse --show-toplevel)" \
     | tr -c '[:alnum:]-' '-' \
     | sed 's/-*$//; s/^-*//'
[ -z "$slug" ] && slug="furrow"
```

Rules: map any non-alnum/non-hyphen char to `-`, strip leading/trailing hyphens,
fall back to literal `furrow` if the result is empty. Guarantees no
`$XDG_STATE_HOME/furrow/-/` or `$XDG_STATE_HOME/furrow//` paths.

```json
{
  "schema_version": 1,
  "repo_slug": "furrow-install-and-merge",
  "repo_root": "/home/jonco/src/furrow-install-and-merge",
  "install_mode": "source|consumer",
  "migration_version": "1.0",
  "installed_at": "2026-04-22T00:00:00Z",
  "last_install_at": "2026-04-22T00:00:00Z",
  "artifacts": {
    "bak_files": [{"path": "bin/alm", "staged_at": "...", "restored_from": "sha"}],
    "symlink_staging": [{"link": ".claude/commands/specialist:x.md", "target": "../../specialists/x.md"}]
  },
  "furrow_version": "0.x.y"
}
```

Field types and requiredness:

| Field | Type | Required | Notes |
|-------|------|----------|-------|
| `schema_version` | integer | yes | Start at 1; bump on schema change. |
| `repo_slug` | string | yes | Normalized per rules above; never empty. |
| `repo_root` | absolute path string | yes | |
| `install_mode` | enum `"source"\|"consumer"` | yes | Set by SOURCE_REPO presence. |
| `migration_version` | **string (semver-like)** | yes | `"0"` = pre-XDG; `"1.0"` = XDG (this wave). Future migrations bump to `"1.1"`, `"2.0"`, etc. Chosen over integer to allow minor version bumps without schema churn. |
| `installed_at` | RFC3339 timestamp | yes | First install. |
| `last_install_at` | RFC3339 timestamp | yes | Updated every run. |
| `artifacts.bak_files` | array | optional (default []) | Provenance for `.bak` quarantine. |
| `artifacts.symlink_staging` | array | optional (default []) | Planned-but-not-yet-created symlinks. |
| `furrow_version` | string | optional | Echo of `frw --version`. |

### `install.sh` / `bin/frw.d/install.sh` — new arguments/branches

- **`install.sh`**: add detection near top (step 1i):
  - `if [ -f .furrow/SOURCE_REPO ]; then INSTALL_MODE=source; else INSTALL_MODE=consumer; fi`
  - `INSTALL_MODE=source`: do NOT copy SOURCE_REPO to target; do NOT bootstrap
    consumer `.gitignore` additions; do regenerate in-repo specialist/rules
    symlinks using `../../specialists/` and `../../.claude/rules/` targets.
  - `INSTALL_MODE=consumer`: existing symlink bootstrap; add `*.bak` + XDG state
    dirs to gitignore if missing (step 1h).
- **`bin/frw.d/install.sh`**: honor `$XDG_STATE_HOME` env (default
  `$HOME/.local/state`). Quarantine target for `.bak` files becomes
  `$XDG_STATE_HOME/furrow/$repo_slug/bak/` (step 1j). Write/update
  `install-state.json` per every invocation.
- **New flag**: `--xdg-state-home <path>` on `bin/frw.d/install.sh` — override for test isolation.

### Sort-by-(created_at, id) — deterministic write

Applies to `.furrow/seeds/seeds.jsonl` and `.furrow/almanac/todos.yaml` on
every CLI-mediated write (step 1k). Grounded in AD-4.

**seeds.jsonl (JSONL)**:
- Algorithm: read all lines → for each, `jq -r '[.created_at, .id] | @tsv'` to
  produce a sortable key → `LC_ALL=C sort -t $'\t' -k1,1 -k2,2` → reassemble
  original lines in the sorted order.
- Invariant: ONE JSON object per line, no pretty-printing, no trailing newline
  beyond the final `\n`. Verified by `jq -c . seeds.jsonl | diff - seeds.jsonl`
  returning identical output.
- Writer: `bin/sds` (and any script that appends to seeds.jsonl) must call
  `frw normalize-seeds` (helper added to `common.sh`, NOT `common-minimal.sh`)
  before exit.

**todos.yaml (YAML)**:
- Algorithm: `LC_ALL=C yq -o=yaml 'sort_by(.created_at, .id)' todos.yaml > tmp && mv tmp todos.yaml`.
- Invariant: list order is `(created_at ASC, id ASC)` bytewise; re-applying the
  sort is a no-op. Round-trip test: `yq -o=yaml . todos.yaml | diff - todos.yaml`.
- Writer: `bin/alm` and any script appending to todos.yaml must call
  `frw normalize-todos` helper.

**LC_ALL=C lock**: every sort invocation is prefixed `LC_ALL=C sort …` to guarantee byte-order sort independent of locale.

**`rws validate-sort-invariant` — verifier CLI** (wave-1 contract; merge-process-skill consumes this in wave 3):

- **Path**: extension of `bin/rws`, new subcommand.
- **CLI**: `rws validate-sort-invariant [--file <path>] [--key <key-expr>]`.
  Default `--file` pair: runs against both `.furrow/seeds/seeds.jsonl` and `.furrow/almanac/todos.yaml`. Passing `--file` runs against a single target.
- **Behavior**: re-applies the sort algorithm in-memory; diffs against on-disk file. Exit 0 if bytewise identical; exit 3 if out-of-order; exit 2 usage.
- **Stderr on fail**: `[furrow:error] sort-invariant violated on <path>: first out-of-order pair at line <n>`.
- **No mutation**: verifier NEVER writes; for repair use `frw normalize-seeds` or `frw normalize-todos`.

### CI contamination-check — `bin/frw.d/scripts/ci-contamination-check.sh`

Landed in step 1m (AD-2, AD-9).

- **Path**: `/bin/frw.d/scripts/ci-contamination-check.sh`.
- **Input**: `git diff --name-only --diff-filter=ACM <base>..HEAD` (base defaults to `origin/main`; override via `--base <ref>`).
- **Banned patterns** (exit 1 if any match):
  - `^bin/(alm|rws|sds)$` where the entry is a symlink in HEAD (`git ls-tree -r HEAD | awk '$1=="120000"'`).
  - `^bin/.*\.bak$`
  - `^\.claude/commands/specialist:.*\.md$` where symlink target (per `git cat-file -p`) escapes the worktree (`../../..` depth OR absolute path).
- **Also runs**: `rescue.sh --baseline-check` → if exit 3, fail the build with stderr `[furrow:error] baseline drift: common-minimal.sh has changed but rescue.sh bundled baseline was not refreshed`.
- **Exit codes**: 0 clean; 1 contamination found; 2 usage error; 3 baseline drift.
- **Wire-up**: add entry to `.furrow/furrow.yaml`:
  ```yaml
  ci_commands:
    - name: contamination-check
      command: bin/frw.d/scripts/ci-contamination-check.sh --base origin/main
  ```

## Acceptance Criteria (Refined)

Each refined AC retains the original letter sequence (A–J for definition ACs 1–10).

- **AC-A (SOURCE_REPO sentinel)** — `/.furrow/SOURCE_REPO` exists and is tracked (`git ls-files .furrow/SOURCE_REPO` returns the path, exit 0); file size is between 100 and 256 bytes; `install.sh` invoked with target-dir differing from repo-root MUST `exit 2` and MUST NOT create `$TARGET/.furrow/SOURCE_REPO`. `bin/frw.d/install.sh` detection branch: with sentinel present, skip step that writes consumer `.gitignore`; branch covered by integration test `test-install-source-mode.sh`.

- **AC-B (specialist symlink repair + validator)** — all 22 `.claude/commands/specialist:*.md` symlinks resolve (via `readlink -f`) to a path beginning with `<repo-root>/specialists/`; additionally, `.claude/rules/cli-mediation.md` and `.claude/rules/step-sequence.md` resolve to `<repo-root>/.claude/rules/` (source-form) or regular files (consumer-form). Install-time validator `bin/frw.d/install.sh` exits 1 with `[furrow:error] unresolved symlink: <path>` if any of the 24 symlinks fail `readlink -e`. Evidence base: `research/r3-boundary-evidence.md §1`.

- **AC-C (XDG state layout)** — no `.bak` files exist under `bin/` or `.claude/rules/` after `install.sh` runs (verified by `find bin .claude/rules -name '*.bak' | wc -l` = 0); all `.bak` files from the install live under `$XDG_STATE_HOME/furrow/$(repo_slug)/bak/`; `install-state.json` exists at `$XDG_STATE_HOME/furrow/$(repo_slug)/install-state.json` and validates against the schema in §Interface Contract. `$XDG_STATE_HOME` override via env MUST be honored (test sets `XDG_STATE_HOME=/tmp/xdg-test` and asserts artifacts land there).

- **AC-D (pre-commit type-change + .bak block)** — `git add` followed by `git commit` of (a) a type-change from regular → symlink on any of `bin/alm`, `bin/rws`, `bin/sds`, `.claude/rules/*.md` OR (b) a staged `bin/*.bak` file exits non-zero with the hook's `[furrow:warning]` on stderr. The SAME commit attempt with `--no-verify` succeeds (git bypass), but the hook body's warning MUST still be reachable when invoked directly (`./.git/hooks/pre-commit` returns 1 with stderr warning).

- **AC-E (merge-conflict-resistant data files)** — after any write, `seeds.jsonl` satisfies: (i) every line is a single JSON object (`jq -c . file | wc -l == wc -l file`); (ii) the file is byte-identical to `LC_ALL=C sort` over `[created_at, id]` keys. `todos.yaml` satisfies: `yq -o=yaml 'sort_by(.created_at, .id)' todos.yaml` produces the same bytes as the file on disk. Idempotency: second `frw normalize-seeds` / `frw normalize-todos` invocation produces zero diff.

- **AC-F (rescue lands first; additive ordering)** — `bin/frw.d/scripts/rescue.sh` exists AND `grep -E '(^|[[:space:]])(\.|source)[[:space:]].*common(-minimal)?\.sh' bin/frw.d/scripts/rescue.sh` returns empty (exit 1). Git log shows commit order: step 1a (SOURCE_REPO) → 1b (symlinks) → 1c (common-minimal.sh) → 1d (rescue.sh) → 1e (rescue tests green) → 1f (common.sh reduction). Re-ordering test: checking out the commit at step 1e and running `tests/integration/test-rescue.sh` MUST succeed BEFORE `common.sh` is reduced in step 1f (enforced by bisect-style CI gate).

- **AC-G (common.sh hook-cascade split)** — `bin/frw.d/lib/common-minimal.sh` exists with exactly the 8 hook-safe functions listed in §Interface Contract (verified by `grep -cE '^[a-z_]+\(\) \{' common-minimal.sh` = 8); all 12 hooks under `bin/frw.d/hooks/` source `common-minimal.sh`, not `common.sh` (verified by `grep -l 'common.sh' bin/frw.d/hooks/ | wc -l` = 0 and `grep -l 'common-minimal.sh' bin/frw.d/hooks/ | wc -l` >= 10). Rationale recorded as an entry in `.furrow/almanac/rationale.yaml` keyed `common-sh-hook-cascade-split` citing AD-1 and `research/r2-commonsh-split.md`.

- **AC-H (legacy-install detection + idempotent upgrade)** — running `install.sh` on a fixture that has `bin/{alm,rws,sds}.bak` and `.claude/furrow.yaml` (legacy pre-XDG markers per `research/r1-install-mutations.md §2`) detects legacy state, moves `.bak` files to `$XDG_STATE_HOME/furrow/$repo_slug/bak/`, writes `install-state.json` with `migration_version: "1.0"`, and removes the legacy `.claude/furrow.yaml` if superseded. Second run on the same fixture produces zero filesystem diff (`diff -r` between snapshots before/after the second run returns empty) AND updates only the `last_install_at` timestamp in `install-state.json`.

- **AC-I (integration tests present and green)** — 5 integration tests exist in `tests/integration/` and exit 0:
  1. `test-install-source-mode.sh` — SOURCE_REPO detection in source repo.
  2. `test-install-consumer-mode.sh` — consumer tempdir branch.
  3. `test-install-xdg-override.sh` — `$XDG_STATE_HOME` override honored.
  4. `test-install-idempotency.sh` — `install.sh` re-run produces zero new changes.
  5. `test-precommit-block.sh` — hooks block type-changes + `.bak` staging.
  6. `test-rescue.sh` — rescue repairs broken `common.sh` in <=2 seconds wall-clock (measured by `time`).
  7. `test-precommit-bypass.sh` — `[furrow:warning]` emitted when hook bypassed.

- **AC-J (self-hosting documentation)** — `/docs/architecture/self-hosting.md` exists with at minimum the H2 sections: `## Source form vs installed form`, `## Path differences`, `## Boundary enforcement points`. Each section contains at least 3 bullet points. Document references: `.furrow/SOURCE_REPO`, `$XDG_STATE_HOME/furrow/`, `bin/frw.d/hooks/pre-commit-*.sh`, `bin/frw.d/scripts/rescue.sh`, `bin/frw.d/scripts/ci-contamination-check.sh`. Verified by `grep -cE '^## (Source form|Path differences|Boundary enforcement)' docs/architecture/self-hosting.md` = 3.

## Test Scenarios

### Scenario: SOURCE_REPO refuse-copy guard
- **Verifies**: AC-A
- **WHEN**: `cp -r ./.furrow /tmp/consumer/.furrow; cd /tmp/consumer; /path/to/install.sh .`
- **THEN**: exit code 2; stderr contains `refusing to copy .furrow/SOURCE_REPO`; `/tmp/consumer/.furrow/SOURCE_REPO` does NOT exist post-run.
- **Verification**: `tests/integration/test-install-consumer-mode.sh` — asserts exit code and inverse existence check.

### Scenario: 24-symlink repair (specialist + rules)
- **Verifies**: AC-B
- **WHEN**: precondition — `.claude/commands/specialist:*.md` symlinks exist with targets matching `../../../furrow/...` (the current contaminated state per `research/r3-boundary-evidence.md §1`). Run `install.sh` in source-mode.
- **THEN**: post — `readlink .claude/commands/specialist:test-engineer.md` resolves to `<repo-root>/specialists/test-engineer.md`; same holds for all 22 specialist symlinks and 2 rules symlinks. `find .claude/commands -name 'specialist:*.md' -type l -exec readlink -f {} \; | grep -v "$PWD/specialists" | wc -l` = 0.
- **Verification**: `tests/integration/test-install-source-mode.sh` asserts 24 symlinks resolve into `<repo>/{specialists,.claude/rules}/`.

### Scenario: XDG state override
- **Verifies**: AC-C
- **WHEN**: `XDG_STATE_HOME=/tmp/xdg-test ./install.sh .`
- **THEN**: `/tmp/xdg-test/furrow/<repo-slug>/install-state.json` exists and validates; no `.bak` files appear under `bin/` or `.claude/rules/`.
- **Verification**: `tests/integration/test-install-xdg-override.sh` — asserts file-existence + schema check via `jq -e 'has("schema_version","install_mode","migration_version")'`.

### Scenario: Idempotent re-run
- **Verifies**: AC-C, AC-H
- **WHEN**: snapshot filesystem (tar), run `install.sh .`, snapshot again, diff snapshots.
- **THEN**: only difference is `last_install_at` timestamp in `install-state.json`; no other file added, modified, or removed.
- **Verification**: `tests/integration/test-install-idempotency.sh`.

### Scenario: Pre-commit type-change block
- **Verifies**: AC-D
- **WHEN**: in a clean tree, `rm bin/alm && ln -s ../bin/sds bin/alm && git add bin/alm && git commit -m 'test'`
- **THEN**: git exits 1; stderr matches `\[furrow:warning\] pre-commit: refusing type-change`; commit not created.
- **Verification**: `tests/integration/test-precommit-block.sh` — assert exit 1 + stderr regex match.

### Scenario: Pre-commit .bak staging block
- **Verifies**: AC-D
- **WHEN**: `touch bin/alm.bak && git add -f bin/alm.bak && git commit -m 'test'`
- **THEN**: git exits 1; stderr matches `refusing to stage install-artifact bin/alm\.bak`.
- **Verification**: `tests/integration/test-precommit-block.sh` (2nd subtest).

### Scenario: seeds.jsonl sort determinism + jq-friendliness
- **Verifies**: AC-E
- **WHEN**: write 10 seeds with shuffled `created_at` timestamps; call `frw normalize-seeds`; read result.
- **THEN**: `jq -c . seeds.jsonl` matches file bytes (no reformatting, one object per line); keys sorted `(created_at, id)` ASC; rerun produces zero diff.
- **Verification**: `tests/integration/test-sort-seeds.sh` — uses `diff <(jq -c . seeds.jsonl) seeds.jsonl` and `cmp` against expected sorted output.

### Scenario: todos.yaml sort determinism
- **Verifies**: AC-E
- **WHEN**: write 10 todos with varied `created_at`; call `frw normalize-todos`; read result.
- **THEN**: `yq -o=yaml 'sort_by(.created_at, .id)' todos.yaml | cmp - todos.yaml` exits 0.
- **Verification**: `tests/integration/test-sort-todos.sh`.

### Scenario: rescue.sh — HEAD path recovery
- **Verifies**: AC-F, AC-I
- **WHEN**: corrupt `bin/frw.d/lib/common.sh` (`echo 'garbage(' >> common.sh` to break `sh -n`); HEAD has a valid version; run `bin/frw.d/scripts/rescue.sh --apply`.
- **THEN**: exit 0; `sh -n bin/frw.d/lib/common.sh` succeeds; file contents equal `git show HEAD:bin/frw.d/lib/common.sh`; wall-clock time <=2 seconds.
- **Verification**: `tests/integration/test-rescue.sh` subtest `head_path`; uses `time -p` and cmp.

### Scenario: rescue.sh — bundled-baseline fallback

## Implementation Notes

### Commit-sequence invariant (Foundation before Enforcement)

Per plan.json `internal_sequencing`, the Foundation pass (1a–1e) MUST land and
its tests MUST be green BEFORE any Enforcement pass commit (1f–1n). Concretely:

- 1a commits `.furrow/SOURCE_REPO` alone.
- 1b commits the 24 symlink repairs alone (or with SOURCE_REPO).
- 1c commits `common-minimal.sh` as a NEW file — `common.sh` UNCHANGED.
- 1d commits `rescue.sh` — standalone.
- 1e commits `tests/integration/test-rescue.sh` — all 4 subtests green.
- Only after 1e passes in CI may 1f (reduce `common.sh`) land.

A bisect-style CI gate MAY be added later; for this row, the human-enforced
gate is sufficient because commit ordering is auditable via `git log`. Rationale:
AD-2 "mitigation commitment" + definition AC 6 ("ordering invariant enforced by
commit order + a smoke test").

### Pattern for `rescue.sh` arg parsing

Mirror `bin/frw.d/scripts/doctor.sh` — POSIX `while [ $# -gt 0 ]; do case "$1" in ... esac; shift; done`.
No `getopts`, no bash arrays, no `[[ … ]]`. First non-flag arg terminates option parsing.

### `LC_ALL=C` env export

Every call site that invokes `sort`, `comm`, `uniq`, or `tr [:...:]` MUST be
preceded by `export LC_ALL=C` in the same shell scope (or prefixed
`LC_ALL=C sort …`). This is the AD-4 lock that guarantees bytewise-stable
sorts independent of user locale.

### jq-friendly JSONL preservation

Never use `jq '.' seeds.jsonl` (pretty-prints). Always `jq -c '…' seeds.jsonl`.
The normalize script materializes sort keys in a side channel (tsv), sorts, and
then re-emits original lines by index — NOT by re-parsing through jq and
re-emitting. This guarantees byte-for-byte preservation of fields whose jq
representation is non-canonical (e.g., `{"a":1,"b":2}` vs `{"b":2,"a":1}`).

### YAML sort

Use `yq -o=yaml 'sort_by(.created_at, .id)' <file> > <tmp> && mv <tmp> <file>`
(yq v4). Do NOT use `--inplace` — atomic rename is safer. Verify yq version at
script start (`yq --version | grep -q ' v4\.'` or exit 1).

### Do NOT modify `bin/frw.d/scripts/launch-phase.sh` in this deliverable

AD-8 is explicit: `launch-phase.sh` appears in file_ownership for read-protection
only. Wave-1 changes touch install-artifact surfaces (`install.sh`,
`frw.d/install.sh`, hooks, lib/). Wave-2 (`worktree-reintegration-summary`) is
the sole modifier. Any edit to `launch-phase.sh` in this wave is a scope violation.

### pre-commit dispatcher pattern

The project's `.git/hooks/pre-commit` is a shim installed by `bin/frw.d/install.sh`
that sources all `bin/frw.d/hooks/pre-commit-*.sh` scripts in alphabetical order
and exits non-zero if any returns non-zero. Add the new hooks by following the
existing `script-guard.sh` file layout (shebang `#!/bin/sh`, `. "$FURROW_ROOT/bin/frw.d/lib/common-minimal.sh"`, function-body-then-dispatch).
