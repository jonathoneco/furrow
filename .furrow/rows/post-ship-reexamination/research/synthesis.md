## Consolidated per-deliverable picture

### D1 ‚Äî observations-schema

**Discriminator idiom decision**: use `allOf` of `if/then` pairs keyed on `const` for both the `kind` discriminator and the `triggered_by.type` discriminator. Not `oneOf`. Rationale:

- `jsonschema` library treats `if/then` as a "strong match" for error reporting; `oneOf` produces a cascade of unrelated-branch failures and is in `WEAK_MATCHES`.
- JSON Schema draft 2020-12 docs explicitly endorse this idiom for discriminated unions.
- Critical gotcha: the `if` clause must include `required: [<discriminator>]` otherwise missing-discriminator entries vacuously match and fire the wrong branch.
- No existing discriminated-union precedent in repo; this schema sets the house style.

**Validation mechanism (MATERIAL CHANGE FROM DEFINITION)**: `bin/frw.d/scripts/validate-todos.sh` does NOT exist. Todos validation is entirely **inline** in `cmd_validate` at `bin/alm:827-937` (`yq -o=json | python3 Draft202012Validator`). So D1's acceptance criterion "create `validate-observations.sh`" is based on a false premise. The correct pattern is: extend `cmd_validate` inline (or split into `cmd_validate_todos` + `cmd_validate_observations` helpers dispatched by a thin `cmd_validate`), preserving the existing `cmd_validate "<path>"` signature that `cmd_add` (bin/alm:198) and `cmd_triage` (bin/alm:523) depend on. **Flagged to user as an AC amendment.**

### D2 ‚Äî alm-observe-cli

**Dispatch pattern**: copy the existing subcommand-group style (`learn`, `rationale` already follow this): one top-level `case` entry at `bin/alm:1498-1514` mapping `observe)` ‚Üí `cmd_observe "$@"`, then `cmd_observe` dispatches verbs to `_observe_<verb>` helpers. Seven helpers: `_observe_add`, `_observe_list`, `_observe_show`, `_observe_resolve`, `_observe_dismiss`, `_observe_activate`, `_observe_on_archive`.

**Pull-model activation**: the template for "iterate .furrow/rows/*/state.json and filter by archived_at" already exists at `bin/rws:169-183` (`find_active_rows`). Copy-paste with filter flipped. Concrete recipe:

```
for _state_file in "${ROWS_DIR}"/*/state.json; do
  _archived="$(jq -r '.archived_at // "null"' "$_state_file" 2>/dev/null)" || continue
  [ "$_archived" = "null" ] && continue
  # $_archived is ISO 8601 with Z; lexicographic compare works iff all rows use UTC Z
  if [ "$_archived" \> "$_baseline_archived" ]; then _count=$((_count+1)); fi
done
```

**TZ-offset caveat**: lexicographic ISO-8601 comparison is correct only when timestamps are normalized (all `Z` UTC, or all same offset). Today the repo consistently uses `Z`; the rest of the codebase already tolerates this assumption. Document the invariant in the schema or a comment.

**Schema constants**: introduce `SCHEMA_FILE_TODOS` and `SCHEMA_FILE_OBSERVATIONS` rather than reusing the existing `SCHEMA_FILE` global; the validation extension should not smuggle observations into the todos pipeline.

**Help**: add one line to the command table at `bin/alm:1476` region with a one-line description.

**Validation-before-write**: `_observe_add` must write to a temp file, run validation, and `mv` atomically only on success ‚Äî mirroring the general `cmd_add` pattern at `bin/alm:198` (which calls `cmd_validate` after writing).
