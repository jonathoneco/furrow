# Research: alm CLI Design (D5)

## Scripts Absorbed

### extract-todo-candidates.sh → `alm extract`
- Standalone "dumb collector" — scans 3 artifact types per row
- Sources: summary.md (Open Questions), learnings.jsonl (unpromoted pitfalls), reviews/*.json (non-pass dimensions)
- Output: JSON array to stdout
- No semantic reasoning — agent layer handles dedup
- Path change: `.work/{name}/` → `.furrow/rows/{name}/`

### validate-todos.sh → `alm validate`
- Schema validation via yq + python3 jsonschema
- Cross-field: unique ID check
- Default path changes: `todos.yaml` → `.furrow/almanac/todos.yaml`
- Schema path: `adapters/shared/schemas/todos.schema.yaml`

### triage-todos.sh → `alm triage`
- Embedded Python for Kahn's algorithm (topological sort)
- Wave assignment: `wave[node] = max(wave[dep] for dep in deps) + 1`
- Conflict detection: glob-to-regex file overlap within same wave
- Output: JSON with todos + graph + conflicts
- Known bug: bare directory globs need `/**` suffix

## Subcommand Architecture

**Decision: Monolithic `bin/alm` script** (~600-800 lines estimated)

```
alm add [--title T] [--context C]     # Create TODO
alm extract [<row>]                    # Extract candidates from row artifacts
alm list [--seed] [--status S]         # List TODOs
alm show <id>                          # Show TODO + linked seed status
alm triage [--full]                    # Generate roadmap.yaml
alm next [--phase N]                   # Generate handoff prompts
alm validate [<path>]                  # Validate todos.yaml
```

## Data Layout

```
.furrow/almanac/
  todos.yaml          # TODO registry (schema-validated)
  roadmap.yaml        # Structured roadmap (generated by alm triage)
  rationale.yaml      # Component rationale (moved from _rationale.yaml)
  learnings/          # Future: promoted learnings across rows
```

## Schema Changes

Add to `adapters/shared/schemas/todos.schema.yaml`:
- `seed_id`: optional string (kebab-case pattern) — links TODO to seed
- `source_type` enum: add `"legacy"` value

## Claude Code Command Wrappers

Commands become thin wrappers:
- `/work-todos --extract` → calls `alm extract`, then agent does dedup reasoning, then `alm validate`
- `/work-todos --new` → agent interactive flow, calls `alm add`
- `/furrow:triage` → calls `alm triage`, agent handles phase grouping reasoning, writes roadmap.yaml
- `/furrow:next` → calls `alm next`, displays output

## Dependencies

Same as current: yq, jq, python3, git. No new dependencies.
